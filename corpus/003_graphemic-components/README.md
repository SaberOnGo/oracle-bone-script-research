# Graphemic Components / 字形构件

English:
This directory will store component records, component variants, and character-component relationships.

Current registers:

- `000_component-registers/001_all-components-index.csv`: accepted project component records.
- `000_component-registers/002_obimd-subcharacter-main-staging.csv`: OBIMD sub-character to main-character staging index with 2,747 dataset relationships.
- `000_component-registers/003_obimd-subcharacter-glyph-staging.csv`: OBIMD sub-character to glyph-code-point staging index with 41,686 dataset relationships.

The OBIMD staging rows preserve useful hierarchy and glyph-code-point evidence, but they are dataset metadata only. They must be cross-checked before becoming formal component, variant, or oracle-character relationships.

简体中文：
本目录将保存构件记录、构件变体和甲骨字-构件关系。

当前索引：

- `000_component-registers/001_all-components-index.csv`：正式本项目构件记录。
- `000_component-registers/002_obimd-subcharacter-main-staging.csv`：OBIMD sub-character 到 main-character 暂存索引，包含 2,747 条数据集关系。
- `000_component-registers/003_obimd-subcharacter-glyph-staging.csv`：OBIMD sub-character 到 glyph code point 暂存索引，包含 41,686 条数据集关系。

OBIMD 暂存行保存了有价值的层级和字形码位证据，但它们只是数据集 metadata。进入正式构件、变体或甲骨单字关系前必须交叉复核。
