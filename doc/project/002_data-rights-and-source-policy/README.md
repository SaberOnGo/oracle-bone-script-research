# Data Rights And Source Policy / 数据权利与来源政策

English:
This repository should not commit rights-unclear scans, paper PDFs, large image sets, or commercial publication extracts. Public GitHub content should prefer metadata, schemas, indexes, rights-clear samples, and source references.

Every data item should answer: where did it come from, what external ID identifies it, what rights status is known, and who reviewed it.

简体中文：
本仓库不应提交权利状态不明确的扫描件、论文 PDF、大规模图片或商业出版物整理文本。公开 GitHub 内容应优先保存 metadata、schema、索引、权利清楚的样例和来源引用。

每条数据都应回答：资料来自哪里、对应哪个外部 ID、权利状态是什么、由谁复核过。

## Rights Status Values / 权利状态值

- `metadata_only_until_verified`: only metadata and references may be public until rights are checked.
- `metadata_only_until_verified`：权利确认前只公开 metadata 和引用。
- `public_domain_verified`: source is verified public domain.
- `public_domain_verified`：来源已确认公版。
- `licensed_for_repository`: source is licensed for repository use.
- `licensed_for_repository`：来源已授权本仓库使用。
- `local_private_only`: may only be stored locally and must not be pushed.
- `local_private_only`：只能本地保存，不得推送。
