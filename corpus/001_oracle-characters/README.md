# Oracle Characters / 甲骨单字

English:
Oracle character records use bucket directories of 100 project IDs each.

Current bucket-level files may contain review manifests before formal character records are created. A `000_hust-obc-promotion-bucket-manifest.csv` file is a reserved-ID review aid only; it does not make the listed `obs-char-*` IDs accepted project characters.

All 16 HUST-OBC candidate buckets now contain candidate packets named `01_candidate-character-packet.json`. These packets make source-marked candidate metadata available under the target bucket paths, but they remain unaccepted review packets until cross-source review.

Simplified Chinese:
甲骨单字记录按每 100 个本项目 ID 一个 bucket 分档。

当前 bucket 级文件可能先保存复核清单，再建立正式单字记录。`000_hust-obc-promotion-bucket-manifest.csv` 只是保留 ID 的复核辅助文件；其中列出的 `obs-char-*` ID 还不是已接受的本项目甲骨单字。

全部 16 个 HUST-OBC 候选 bucket 现在都包含名为 `01_candidate-character-packet.json` 的候选资料包。这些资料包把带来源标记的候选 metadata 放入目标 bucket 路径，但在跨来源复核前仍只是未接受的复核资料包。
